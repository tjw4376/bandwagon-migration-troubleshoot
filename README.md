## 搬瓦工机房迁移功能是什么

搬瓦工（BandwagonHost）的 KiwiVM 后台自带一个叫"Migrate to another DC"的功能，可以把你的 VPS 从当前机房直接搬到另一个机房——**数据全程保留**，IP 会换，但 VPS 里的文件、配置、系统一点不动。换机房本身不收费，也不限次数。

这是搬瓦工做了十几年总结下来最被用户认可的功能之一，其他家很少能做到这种自由度。搬瓦工目前全球有 25 个以上机房，理论上可以在这些机房之间随意切换（具体能换哪些，取决于你买的套餐）。

问题是，不是所有人都能顺利用上这个功能。

## 搬瓦工无法迁移的五种情况，逐条说清楚

### 情况一：Plan does not support migrations (733102)

这是最直接的。

你的套餐本身就不支持机房迁移。点进迁移页面，直接显示这个错误，什么操作都做不了。

哪些套餐会触发这个错误？

- **香港 CN2 GIA 套餐（HKHK_8）**：香港机房套餐整体不支持后台自助迁移，这是官方规定
- **部分老套餐**：比如早年的"LOS ANGELES - CHINA DIRECT ROUTE"系列直连套餐，购买时没有写明支持迁移的，统一不支持
- **部分限量版套餐**：日本软银限量版、MINICHICKEN 等，机房固定不能换

怎么确认自己的套餐是否支持迁移？在套餐描述里找这句话：**"Automatic migration between datacenters"**。有这句话的可以迁，没有的就不行。

遇到 733102 的唯一出路：要么接受当前机房，要么在后台把套餐升级到支持迁移的系列。搬瓦工后台支持自助升级，从日本软银限量版可以直接升级到 CN2 GIA-E 常规套餐。

### 情况二：Migration system is currently not available (734151)

系统临时不可用。

这个大概率不是你的问题——搬瓦工的迁移后台偶尔会有短暂拥堵或维护，或者你迁移太频繁触发了临时限制。通常等 15-30 分钟再试就好了。

实测下来，90% 的 734151 等一会儿自然解决。真的等了半小时还不行，试试换一个目标机房重新发起迁移，往往能绕过去。再不行就提交工单，客服可以手动处理。

### 情况三：Migration backend is currently not available for this VPS (734152)

这个稍微麻烦一些。

名字和 734151 很像，但原因不同。**最高发的触发原因是 IP 被封**。

搬瓦工的迁移机制需要控制面板和你的 VPS 之间建立通信来完成迁移前的准备。IP 被封之后，这个通信渠道断了，后台发不出迁移指令，就报这个错。

判断 IP 是不是真的被封：用多地 ping 工具测一下你的 VPS IP，如果中国大陆所有地区都 ping 不通，但海外节点能 ping 通，基本就是被封了。搬瓦工 KiwiVM 后台也有官方的 blacklist check 工具，登录后台可以用。

IP 被封之后，无法免费换 IP，也无法直接迁移机房。需要先付费更换一个新 IP，等 IP 恢复正常之后迁移功能才会回来。这是目前绕不开的。

IP 被封。没别的办法。

### 情况四：Failed to migrate (734104 / 734105)

目标机房已满或没有可用 IP。

734104 表示目标机房当前容量满了，734105 是目标机房暂时没有可用 IP 地址。

这两个都是临时状态，等几个小时或者换一个目标机房试试，一般能解决。热门机房（比如 DC9 CN2 GIA、日本东京）偶尔会出现这个情况，因为用的人多。

### 情况五：迁移完成但连不上

不是迁移失败，是迁移成功了但 VPS 暂时处于初始化网络的过程中。

新机房网络初始化需要几分钟到十几分钟不等。还有一种情况是 IP 配置没有自动更新，VPS 跑着但网络不通。

遇到这种情况：先登录 KiwiVM 后台里的 VNC，看 VPS 本身是不是在正常运行。如果 VPS 运行正常，等几分钟再连。如果 VNC 里看到网络相关报错，就针对性查一下 IP 配置问题。

## 迁移操作的完整步骤

买对了套餐，迁移其实很简单。四步搞定：

1. 登录 KiwiVM 控制面板（进搬瓦工后台 → Services → My Services → 找到对应 VPS → 点 KiwiVM Control Panel）
2. 左侧菜单点"Migrate to another DC"
3. 页面会显示当前机房，下方选择目标机房
4. 点"Confirm on next step"，确认后开始迁移

迁移过程中 VPS 会短暂停机，一般几秒到几分钟，数据全保留。迁移完成后会收到邮件通知，也可以在迁移页面看进度。

有一点要注意：**迁移之后 IP 会变**。如果你的 VPS 上跑着网站，要记得更新 DNS 解析。如果服务绑定了静态 IP 地址（而不是 0.0.0.0），还需要修改对应的配置文件。

## 换机房之后连不上，该先查什么

换完机房发现 SSH 连不上，不要慌，按这个顺序排查：

首先，等几分钟。网络初始化要时间，新机房分配好 IP 之后需要一小段时间才稳定。

其次，登录 KiwiVM 里的 VNC，看 VPS 本身在不在线。VNC 能进说明 VPS 是正常运行的，问题出在网络层。

如果 VNC 里看到类似"A start job is running for Raise network interfaces"这种提示，是网络接口在初始化，继续等就行。

如果 VPS 里的 IP 配置没有自动更新，会显示还是旧 IP，这时候需要手动修改网络配置文件，把 IP 改成新机房分配的那个。

## 常见问题

**Q：搬瓦工换机房要收费吗？**

不收费。迁移功能本身完全免费，不限次数。但迁移会消耗流量，因为数据要在机房之间传输一遍。

**Q：换机房之后数据会丢失吗？**

不会。这是搬瓦工迁移功能的核心卖点，VPS 里所有数据完整保留，就像搬了个地方继续跑。

**Q：迁移失败了，VPS 上的数据还在吗？**

在。迁移失败的时候 VPS 会保持在原来的机房，数据不受影响。

**Q：为什么我的套餐不显示迁移选项？**

要么套餐本身不支持（733102 那种情况），要么 IP 被封了导致功能不可用。建议先检查 VPS IP 的可连通性，再确认套餐说明里有没有"Automatic migration between datacenters"这句话。
